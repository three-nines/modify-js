    Function wrapTry

    • wrapTry<T, E, R>(
          logic: () => T | Promise<T>,
          cb_err?: (err: E, partialResult: undefined) => R | Promise<R>,
          cb_last?: (result: T | undefined) => void | Promise<void>,
          isThrow?: boolean,
      ): Promise<T | R | undefined>

      An asynchronous try-catch-finally wrapper for streamlined control flow.

      Type Parameters

      • T

        The expected resolve type of the primary logic function.

      • E

        The expected error type caught in the catch block. Defaults to unknown.

      • R

        The return type of the error callback function (recovery value).

      Parameters

      • logic: () => T | Promise<T>

        The core function to execute safely. Can be synchronous or asynchronous.

      • Optionalcb_err: (err: E, partialResult: undefined) => R | Promise<R>

        Optional error callback. Note: partialResult will always be undefined because if logic throws, its assignment never completes.

      • Optionalcb_last: (result: T | undefined) => void | Promise<void>

        Optional finalizer callback (runs like a finally block). Receives the successful result T from logic, or undefined if logic threw an exception (even if cb_err recovers a value).

      • OptionalisThrow: boolean

        If true, re-throws the caught error after executing all callbacks.

      Returns Promise<T | R | undefined>

      A promise that resolves to:

      • The result of logic (T) on success.
      • The fallback result of cb_err (R) if an error was caught and handled.
      • undefined if an error occurred, isThrow is false, and no cb_err handler was provided.

      Remarks

      • Seamlessly handles both synchronous and asynchronous functions by safely awaiting execution blocks.
      • Guarantees sequential, chronological execution of error hooks (cb_err) and finalizers (cb_last).

      Throws

      If the provided logic argument is not an invocable function.

      Throws

      Re-throws the original caught error if isThrow is true and an exception occurs.

      Example

      const data = await wrapTry(
          async () => await fetchUser(1),
          (err) => {
              console.error("Failed to fetch:", err);
              return { guest: true }; // Fallback value (Type R)
          },
          (primaryResult) => console.log("Operation finished. Was successful:", !!primaryResult),
          false
      );
